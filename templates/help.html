Player UI

You have 3 main sections, Browse, Now Playing and Playlist

BROWSE section

  • Here you have access to all your Local Music, stored on USB or NAS and WebRadios
  • You can type and search in real time
  • To play a track just double click on it (works only on PC)

On the right side of each item you find the extended enqueue menu

  • Add: adds a song to play queue
  • Add and play: adds a song to play queue and plays it immediatly
  • Add, replace and play:replaces current play queue with the item you select and plays it immediatly
  • Update this folder: refreshes the list of files in a specific folder. Useful if you copy new tracks or albums to a folder.
  • Please note that this is the only control available when using mobile browsers, since double click is not available with touch screens.

PLAYBACK section

Elapsed time indicator

  • Repeat all the files on playlist
  • Random play on the queue
  • Repeat the single song
  • Consume mode: delete songs once played

Previous/next song, stop and play/pause

  • They do indicate current playing track and its bitrate/bitdepth
  • When RaspyFi is playing, the play button is showed
  • When RaspyFi is paused, the pause button is showed

Volume knob

  • You can scroll, insert a numeric value or just interact like a good old analog knob
  • Please note that you should activate the hardware or software mixer in the MPD Settings section
  • Mute, if you really need silence

PLAYLIST section

This is your current play queue, currently RaspyFi supports only this playlist, you can remove tracks and search them here

Database Setup

RaspyFi lists all your Music Library in an internal Database. Tracks and Folders have to be scanned to be added to the Database.

You can use a Local Storage (USB HD or Thumb Drive), which is automatically detected. Once you connect it to USB, remember to update the DB via the "Update MPD Database" button.

If you wish to use a Network Attached Storage, you can add as many Shares as you want. Please note that only Samba Shares are allowed with RaspyFi.

To add a network source, just fill the form, save your Network Mount and click "Update MPD Databe" . A green check will appear if everything is all right.
If your network mount isn't properly recognized, try to put a random username on it or reboot the system. To view your files on the BROWSE tab, just refresh the page on your browser.

MPD setup

RaspyFi's core is MPD, a music player daemon that does an excellent job in delivering pure bit perfect pleasure. Here you have some handy features to customize its behaviour. If you're in the mood of manually editing, well you can just edit the /etc/mpd.conf file.

MPD setup

  • Port: default is 6600. Be aware, if edited this settings may result in unreachable player.
  • Gapless mp3 playback: immediatly starts the next song when the previous ends
  • Auto-update: when you add some new files to your music collection, the database is updated automatically
  • Volume control mixer:

DISABLED: No remote volume control available, preserves audio quality but harms couch potatoes
SOFTWARE: Works with every DAC available, but affectes audio quality
HARDWARE: Works only with some DACS (and you probably have to edit the alsa section of mpd.conf) but it enables volume control without affecting sound quality.

Network Setup

Here you can configure your network settings, connecting via wi-fi and assign a static ip to your RaspyFi device. You can also find out which IP it is currently assigned to.

  • IP address: the ip you want to be permanently assigned to RaspyFi
  • Netmask: usually 255.255.255.0 should be good
  • Gateway: the IP of your router. Usually 192.168.1.1
  • DNS: You can set up alternative Domain Name Servers, could be useful.

Wi-Fi config

  • SSID: The name of wireless network you want to connect to
  • Security: WEP (you really sure?) or WPA
  • Passphrase: the passphrase of your Wi-Fi Network

If everything is ok, you'll see an ip assigned to the Wi-Fi section.

AIRPORT/AIRPLAY

You can use RaspyFi also as an Airplay receiver. You can then play in high quality all the music you have in your pc, smartphone or tablet as you do with a remote wireless speaker.

ATTENTION: You must first stop RaspyFi from playing before activating it as an Airport receiver. Then you must stop your Airport playback before restarting your local play mode.

  • iOs: when playing, in the lower right corner tick the Airplay trigger and select RaspyFi.
  • Itunes: when playing, upper left corner. Tick the Airplay trigger and select RaspyFi.
  • Android: you can use AirAudio

RaspyFi Doesn't Boot

Check PSU. It must be at least 1A and between 4,75 and 5,25 volts. Power cord must not be more than 1m long. If using USB powered hard drives, a powered USB hub is needed. Try to download again the image and reflash the SD card.

I cannot hear any sound

RaspyFi outputs the sound automatically. If you have an USB DAC attached, it will output to USB DAC (see compatibility list ). If no USB DAC but HDMI is connected, it will output to HDMI. If nothing else than Analog Jack is plugged, well...

I can't find out the IP of RaspyFi and I can't connect typing raspyfi.local on my browser

  • If you are on windows download and install this
  • From Windows, go on Network and right-click on RaspyFi then "Open device web page"
  • On some Android Devices only typing the right IP will result in connecting to the WEBUI, use methods above to find the right IP

It's sounding crackling and distorted

  • Wait for the Database update to finish
  • If you are accessing network files, go to Database Settings, advanced settings and try gradually decreasing default rsize and wsize values
  • Check your PSU as described above and try disconnecting other devices from the PI.

If all the above doesn't solve the situation, you can get help on the FORUM
